	c. What are mutating errors? How to avoid?
		The basic reason for this error is the way Oracle manages a read consistent view of data. The error is encountered 
		when a row-level trigger accesses the same table on which it is based, while executing. The table is said to be mutating. 
		Mutation will not occur if a single record is inserted in the table (using VALUES clause). If bulk insertion is done or 
		data is inserted from another table mutation will occur.
		
		We can override this problem using temporary Table Concept.
